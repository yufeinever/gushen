# 外部项目对标基线

本项目当前采用“先补数据与闸门，再升级 Agent 编排”的路线。外部项目分工如下：

| 项目 | 适合借鉴的部分 | 当前执行方式 |
| --- | --- | --- |
| TradingAgents | 多分析师、研究经理、交易员、风险辩论、组合经理、记忆复盘 | 已作为主对标矩阵；本轮补 SectorThemeAgent、FundFlowAgent 和数据质量闸门 |
| Microsoft Qlib | 因子工程、数据集管理、回测评估 | 后续把当前 CSV 数据集升级为可回测因子面板 |
| FinGPT / FinRobot | 金融 NLP、新闻/社交情绪、金融问答 Agent | 后续用于 A股版 SentimentNarrativeAgent 的论坛、公告和新闻语义判断 |

## 当前原则

- 国内行情、公告、板块和资金数据优先直连，不主动走代理。
- 外部接口失败时，必须标记 `fallback` 或 `missing`，不能伪装成真实数据。
- 数据质量闸门未通过时，只能输出研究观察，不能输出真实买入/卖出建议。
- 对标分只反映真实执行能力；类名相似但没有数据和决策影响，不算深度对标。
