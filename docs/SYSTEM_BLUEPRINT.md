# 系统蓝图

## 目标

构建一个 A 股高流动性股票的研究、信号、风控和复盘系统。系统输出的是“可审计的交易建议和风险说明”，不是黑箱自动交易。

## 核心假设

- 成交额前 100 的股票流动性更好，价格冲击和异常成交噪声相对更低。
- A 股短中线交易必须显式处理 T+1、涨跌停、停牌、集合竞价、公告风险和交易费用。
- AI 在文本理解和假设生成上有优势，但最终信号必须回到结构化数据、回测和风控。

## 数据层

第一阶段建议通过 AKShare 接入：

- 日线行情：开高低收、成交量、成交额、复权因子。
- 股票基础信息：代码、名称、交易所、板块、上市日期、ST/退市风险状态。
- 交易日历：交易日、节假日、临时休市。
- ST、停复牌、涨跌停池。
- 公告数据：定期报告、重大事项、风险提示。
- 财务摘要：营收、利润、现金流、资产负债、ROE。

第二阶段再考虑：

- 分钟行情。
- 龙虎榜、融资融券、北向资金、股东户数。
- 行业指数、概念标签、新闻情绪。

## 股票池层

每日收盘后生成：

- `universe_date`
- `rank_by_amount`
- `ts_code`
- `name`
- `exchange`
- `amount`
- `close`
- `pct_change`
- `turnover_rate`
- `is_st`
- `is_suspended`
- `limit_status`

默认过滤：

- 剔除停牌。
- 剔除退市整理期。
- 剔除无法稳定取得历史数据的标的。
- ST 股票默认不参与第一阶段策略。

## 特征层

第一版特征：

- 趋势：5/10/20/60 日均线、均线斜率、价格相对均线距离。
- 动量：1/3/5/10/20 日收益率。
- 成交：成交额排名、成交额变化率、量比、换手率。
- 波动：ATR、日内振幅、近 20 日波动率。
- 位置：近 20/60 日高低点分位。
- 风险：涨跌停距离、连续涨跌停、缺口、异常波动。

## 策略研究层

第一批策略只做研究：

- 高成交额趋势延续。
- 放量突破后短期动量。
- 高流动性回调反转。
- 行业内强弱轮动。
- 公告事件后的风险回避。

每个策略必须输出：

- 样本区间。
- 股票池规则。
- 入场条件。
- 出场条件。
- 仓位规则。
- 交易费用、滑点和涨跌停处理。
- 收益、回撤、胜率、盈亏比、换手、最大连续亏损。

## AI 层

AI 输入必须结构化：

- 股票池快照。
- 特征表。
- 策略信号表。
- 公告和新闻摘要。
- 近期复盘记录。

AI 输出必须可落表：

- `candidate_code`
- `reason`
- `supporting_features`
- `risk_factors`
- `invalid_condition`
- `suggested_action`
- `confidence_note`

其中 `suggested_action` 只能是 `observe`、`research`、`paper_trade`、`avoid`，第一阶段不输出 `buy` 或 `sell`。

## 执行层

阶段划分：

1. 离线研究：只跑历史数据。
2. 每日观察：收盘后生成候选观察清单。
3. 模拟盘：人工确认，不接真实资金。
4. 小资金实盘：仍需人工确认。
5. 半自动执行：只有在长期稳定、风控完整、审计日志完整后讨论。

## 技术建议

后端优先选择 Python：

- 数据处理：`pandas`、`numpy`、`polars`。
- 存储：先用 `duckdb` 或 `sqlite`，以后再迁移 PostgreSQL。
- 回测：先自建简洁事件驱动框架，等规则稳定后再评估第三方框架。
- 任务调度：先用本地脚本和计划任务，后续再接 Airflow/Prefect。

前端暂缓，等数据和回测跑通后再做仪表盘。
